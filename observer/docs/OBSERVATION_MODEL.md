# Observation Model

## Purpose

An Observation represents something detected by an adapter.

It is raw information.

It is not engineering evidence.

It is not an AIEA Event.

The Observer Core decides whether an Observation becomes engineering evidence.

---

# Observation

Every Observation contains:

- id
- timestamp
- source
- type
- payload

---

## id

Unique identifier generated by the adapter.

Example

```
OBS-000001
```

---

## timestamp

UTC timestamp.

Example

```
2026-06-29T08:35:42Z
```

---

## source

Where the observation came from.

Examples

- vscode
- git
- terminal
- copilot
- codex
- mcp

---

## type

A normalized identifier.

Examples

```
file.saved

file.created

file.deleted

terminal.command

terminal.finished

git.commit

git.checkout

copilot.suggestion

copilot.accepted

copilot.rejected

codex.request

codex.response
```

---

## payload

Raw data provided by the adapter.

The Observer Core is responsible for interpreting it.

Example

```json
{
  "file": "PlayerService.cs",
  "language": "csharp"
}
```

---

# Adapter Responsibility

Adapters only observe.

Adapters never:

- create engineering events
- modify activities
- generate reports

Adapters emit Observations.

---

# Observer Responsibility

Observer receives Observations.

It applies rules.

It decides whether they become:

- AIInteraction
- Finding
- Decision
- Risk
- Issue
- Milestone

or are ignored.

---

# Golden Rule

Adapters detect.

Observer interprets.

Core records.
